Before writing a single analysis, you need to know exactly what you have. Run each cell, read the output carefully, and understand what each table represents. This is what every real analyst does first.

Know Your Tables First
olist_orders_dataset
~99,441 rows · CORE TABLE
Every order ever placed. Has order status, purchase timestamp, delivery dates. This is the spine of everything.
olist_order_items_dataset
~112,650 rows
What was in each order — product, seller, price, freight value. One order can have multiple items.
olist_customers_dataset
~99,441 rows
Customer info — unique ID, city, state. Note: one real person can have multiple customer IDs.
olist_order_payments_dataset
~103,886 rows
Payment method, installments, payment value. One order can have split payments.
olist_order_reviews_dataset
~99,224 rows
Customer review scores (1–5) and comments after delivery. Goldmine for sentiment analysis.
olist_products_dataset
~32,951 rows
Product details — category, dimensions, weight, photos count. Category names are in Portuguese.
olist_sellers_dataset
~3,095 rows
Seller location data — city and state. Useful for geographic analysis.
olist_geolocation_dataset
~1,000,163 rows
ZIP code to lat/lng mapping. Used for mapping customer + seller locations. Largest file.
product_category_name_translation
~71 rows
Translates Portuguese category names to English. Always join this to products table.

CELL 01
Import Libraries & Load All Tables
Run First
We load all 9 CSVs into a Python dictionary so we can access any table by name. This is cleaner than creating 9 separate variables.
import pandas as pd
import numpy as np
import os

# Path to your raw data folder
DATA_PATH = '../data/raw/'

# Load all CSVs into a dictionary
tables = {}
for file in os.listdir(DATA_PATH):
    if file.endswith('.csv'):
        name = file.replace('olist_', '').replace('_dataset.csv', '').replace('.csv', '')
        tables[name] = pd.read_csv(DATA_PATH + file)

# See what we loaded
for name, df in tables.items():
    print(f"{name:<40} {len(df):>8,} rows  |  {len(df.columns)} columns")

CELL 02
Check for Nulls in Every Table
Understand Output
Nulls = missing data. In real jobs, ignoring nulls causes wrong analysis. We need to know exactly where they are before we do anything else.
# Check null percentage in every table
for name, df in tables.items():
    null_pct = (df.isnull().sum() / len(df) * 100).round(1)
    has_nulls = null_pct[null_pct > 0]
    if len(has_nulls) > 0:
        print(f"\n--- {name} ---")
        print(has_nulls.to_string())

CELL 04
Monthly Revenue Trend (First Real Insight)
Your First Chart
Let's join orders + payments and plot monthly revenue. This is your first real business insight from the data. Every data analyst interview asks "what did you find interesting in your project?" — this chart is your answer.
import matplotlib.pyplot as plt

payments = tables['order_payments']

# Join orders with payments
orders_pay = orders.merge(payments, on='order_id', how='inner')

# Keep only delivered orders
orders_pay = orders_pay[orders_pay['order_status'] == 'delivered']

# Extract year-month
orders_pay['month'] = orders_pay['order_purchase_timestamp'].dt.to_period('M')

# Monthly revenue
monthly_rev = orders_pay.groupby('month')['payment_value'].sum().reset_index()
monthly_rev['month'] = monthly_rev['month'].astype('str')

# Plot it
plt.figure(figsize=(14, 5))
plt.plot(monthly_rev['month'], monthly_rev['payment_value'],
         color='#00ff88', linewidth=2, marker='o', markersize=4)
plt.fill_between(range(len(monthly_rev)), monthly_rev['payment_value'],
                 alpha=0.1, color='#00ff88')
plt.xticks(rotation=45, ha='right', fontsize=8)
plt.title('Monthly Revenue — Olist E-Commerce (2016–2018)', fontsize=13, pad=15)
plt.ylabel('Revenue (BRL)')
plt.xlabel('Month')
plt.tight_layout()
plt.savefig('../docs/monthly_revenue.png', dpi=150)
plt.show()
print(f"Peak month revenue: R$ {monthly_rev['payment_value'].max():,.0f}")
print(f"Total revenue: R$ {monthly_rev['payment_value'].sum():,.0f}")
What you'll see: A strong upward trend from 2016 → 2018 with a massive spike around Black Friday (Nov 2017). This is your first real finding — growth story + seasonality. Screenshot this chart, it's going in your README.
CELL 05
Top Product Categories by Revenue
Business Insight
Which product categories make the most money? This is a classic business question. We need to join 4 tables to answer it — orders → items → products → translation. This is where SQL thinking kicks in.
items     = tables['order_items']
products  = tables['products']
trans     = tables['product_category_name_translation']

# Join all 4 tables
merged = (orders_pay
    .merge(items, on='order_id')
    .merge(products, on='product_id')
    .merge(trans, on='product_category_name', how='left')
)

# Revenue by English category name
cat_rev = (merged
    .groupby('product_category_name_english')['payment_value']
    .sum()
    .sort_values(ascending=False)
    .head(10)
    .reset_index()
)

# Plot horizontal bar chart
plt.figure(figsize=(10, 6))
plt.barh(cat_rev['product_category_name_english'],
         cat_rev['payment_value'], color='#60aaff')
plt.xlabel('Total Revenue (BRL)')
plt.title('Top 10 Product Categories by Revenue', pad=15)
plt.gca().invert_yaxis()
plt.tight_layout()
plt.savefig('../docs/top_categories.png', dpi=150)
plt.show()
print(cat_rev)
What you'll find: Health & Beauty, Watches & Gifts, Bed/Bath/Table are top earners. This multi-table join is exactly what SQL interviews test. You just did it in Pandas — tomorrow we'll write the same logic as a SQL query.
